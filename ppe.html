Projet NutriFit — Assistant nutritionnel pour sportifs

Description du projet :

NutriFit est une application web locale, sans backend, destinée aux clients d’une salle de sport. Elle permet de rechercher des produits alimentaires, d’afficher leurs valeurs nutritionnelles, et de recevoir des recommandations personnalisées selon le profil sportif de l’utilisateur (prise de masse, sèche, entretien). Ce projet a été réalisé intégralement seul, combinant à la fois le développement logiciel (SLAM) et la configuration système (SISR).

Missions et technologies utilisées :

  • Développement frontend : HTML/CSS responsive avec Bootstrap, JavaScript (DOM, événements, localStorage), intégration de l’API Open Food Facts (requêtes fetch et traitement JSON), gestion du profil utilisateur et historique local.
  • Configuration système et déploiement : mise en place d’un poste local en mode kiosque (Debian/Windows), déploiement de l’application avec Apache/Nginx, automatisation de la supervision, sécurisation du poste (pare-feu, restrictions).

Compétences développées :

  • Maîtrise des appels API REST et manipulation de JSON en JavaScript.
  • Création d’interfaces dynamiques et réactives.
  • Utilisation avancée du localStorage pour gestion des données.
  • Déploiement, configuration et sécurisation d’une application web locale.
  • Autonomie dans la réalisation d’un projet alliant développement et infrastructure.
  • Rédaction de documentation technique et utilisateur.

Projet 2 — Intégration et déploiement d’une solution serveur sur le réseau

Contexte général

Intégration dans l’équipe administration systèmes et réseaux pour déployer et maintenir une solution serveur permettant la gestion de connexions clients et la diffusion de données en temps réel.

Objectifs et Tâches à réaliser

Le projet consiste à :

  • Cloner un dépôt GitHub contenant le serveur
  • Lancer le serveur en local sur la machine
  • Tester la connexion client au serveur
  • Modifier et mettre à jour la documentation README
  • Ouvrir les ports nécessaires sur le firewall
  • Créer un service systemd pour démarrer automatiquement le serveur
  • Dockeriser la solution pour faciliter le déploiement
  • Déployer la solution sur un serveur distant
  • Superviser et dépanner en cas de problème

Dépôt GitHub du projet : https://github.com/JulienMarcucci/Classcord-server

Plan détaillé du déroulement

Jour 1 - Lundi : Prise en main du projet

  • Cloner le dépôt GitHub.
  • Compiler et lancer le serveur localement.
  • Tester la connexion client.
  • Analyser et compléter la documentation (README).

Jour 2 - Mardi : Déploiement et automatisation

  • Configurer le firewall et ouvrir le port du serveur.
  • Créer un service systemd pour automatiser le démarrage.
  • Écrire un Dockerfile et créer une image Docker.

Jour 3 - Mercredi : Déploiement sur serveur distant

  • Déployer le container Docker sur un serveur distant.
  • Configurer la supervision et les alertes.

Jour 4 - Jeudi : Tests, dépannage et optimisations

  • Tester le service en condition réelle.
  • Corriger les erreurs et optimiser la configuration.

Jour 5 - Vendredi : Bilan et présentation finale

  • Préparer la présentation finale du projet.
  • Réaliser un bilan des tâches effectuées.
  • Effectuer une démonstration devant l’équipe.
  • Collecter les retours et préparer la documentation finale.

Compétences mobilisées

  • Concevoir une solution d’infrastructure réseau.
  • Installer, tester et déployer une solution.
  • Exploiter, dépanner et superviser un service réseau.
